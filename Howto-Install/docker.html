My Docker notes

If you want to simulate real world containers infrastructure you could deploy an OS on Qemu and start container within it.

Advantage of Docker containers Versus a small Qemu VM

Containers use their host kernel which means they consume less RAM and CPU power. Docker container images are also available from a extensive registry (Docker hub), you can fork them in order to add your own tools and create custom containers.

Docker containers are not real VM

To make it clear even if in GNS3 we use container to emulate computers it's not the case in real world. Containers use your kernel and run a single program not the whole operating system.

https://www.gns3.com/support/docs/docker-2

Devuan, Debian (Docker official archive mirror)

installation steps

  • On Debian Jessie 
    deb https://apt.dockerproject.org/repo debian-jessie main
  • On Debian Stretch/Sid 
    deb https://apt.dockerproject.org/repo debian-stretch main
  • Update the APT package index and verify that APT is pulling from the right repository. 
    sudo apt-get update
sudo apt-get install apt-transport-https ca-certificates  
sudo apt-key adv --keyserver hkp://p80.pool.sks-keyservers.net:80 --recv-keys 58118E89F3A912897C070ADBF76221572C52609D  
sudo apt-cache policy docker-engine
  • Install Docker 
    sudo aptitude install docker-engine
sudo adduser cloud docker
docker run hello-world
